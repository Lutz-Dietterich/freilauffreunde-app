## 3. Design und Benutzererfahrung

- **Designstil:** Der Designstil soll modern und ansprechend sein. Es wird ein minimalistisches Design bevorzugt.
- **Benutzerfreundlichkeit:** Die Website soll einfach zu bedienen sein. Die Navigation soll intuitiv sein.
- **Responsive Design:** Es wird ein responsives Design verwendet. Die Website soll auf allen Geräten gut aussehen. Es sollen drei Breakpoints verwendet werden. Mobile, Tablet und Desktop. Die Breakpoints sollen bei 480px, 768px und 1024px liegen.

## 4. Technische Anforderungen

- **Content-Management-System (CMS):** Es soll ein CMS verwendet werden. Zuerst wird Next.js verwendet. Später wird ein eigenes CMS entwickelt.
- **Hosting:** Im ersten Schritt wird die App auf Vercel gehostet. Spöter wird die App auf einem eigenen Server gehostet.
- **Domain:** Das hosting wird über die Domain `freilauffreunde.de` laufen. Und ist schon bei Strato registriert.
- **Sicherheit:** Ein SSL Zertifikat ist über Vercel schon vorhanden. Später wird ein eigenes Zertifikat benötigt und wird über NGINX konfiguriert.
- **Suchmaschinenoptimierung (SEO):** Es wird Backlinks zu sozialen Medien geben. Die Website wird auf Google und Bing eingetragen.
  Keywords werden in den Texten verwendet. Metatags werden auf bestimmten Seiten dynamisch generiert.
  z.B. Für die Druckdateien werden die Metatags in der Datenbank gespeichert und bei der Generierung der Seite ausgelesen.
- **Performance:** Die Website wird auf Performance getestet. Es wird ein Lighthouse Test durchgeführt.


## 5. Budget und Zeitplan

- **Budget:** Das Budget soll so gering wie möglich gehalten werden und wird durch Eigenleistung erbracht. Das Die Domain kostet 12 Euro im Jahr bei Strato. Das Hosting kostet 0 Euro im Jahr bei Vercel. Später wird ein Server bei Hetzner für 8,09 Euro im Monat gemietet. CX32 mit 8 GB RAM und 80 GB SSD.
- **Zeitrahmen:** Das Projekt soll in 6 Monaten abgeschlossen sein. Am 20.08.2025 geht die Website online. Und wird dann laufend aktualisiert.
- **Zeitplan:** [zeitplan](zeitplan.md)

## 6. Sonstige Anforderungen

- **Barrierefreiheit:** Die Barrierefreiheit soll berücksichtigt werden. Es soll eine gute Kontrastierung der Farben geben. Es soll eine alternative Textbeschreibung für Bilder geben. Die HTML-Struktur soll semantisch korrekt sein. Es soll eine Tastaturnavigation möglich sein. Es soll eine gute Lesbarkeit der Schriftgröße geben. Es soll eine gute Lesbarkeit der Schriftart geben.
- **Sprachen:** Die Webseite soll in Deutsch verfügbar sein.
- **Integrationen:** Es soll Youtube Videos eingebunden werden. Es soll ein Kontaktformular mit Nodemailer eingebunden werden. Es soll ein Newsletter eingebunden werden.Weiter soll eine Instagram Integration eingebunden werden. Sowie Amazon Affiliate Links.
- **Rechtliche Anforderungen:** Es soll ein Impressum und eine Datenschutzerklärung vorhanden sein. Es soll ein Cookie-Banner eingebunden werden. Die Website soll DSGVO-konform sein. Die Datenschutzerklärung soll von mit eRecht24 erstellt werden. Die Cookie-Banner wird falls nötig selbst erstellt.
